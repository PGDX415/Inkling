# 拾光 (Inkling) - 项目说明

## App 名称
- **中文名**：拾光
- **英文名**：Inkling
- 命名意象：拾起时光碎片；Inkling 兼具"一点墨水"与"隐约感受"之意，呼应写日记的初心
- 风格定位：复古文艺，克制、温润、有质感

## 项目概述
一款极简、纯粹的日记本 App，运行于 iPhone / iPad / Mac，使用 SwiftUI 原生开发，一套代码多平台适配。核心目标是提供一个专注、可靠、隐私安全的写日记体验。**不包含图片功能**，专注纯文字记录。

## 多语言（国际化）要求
- **App 从第一版起就支持中文（简体）和英文双语**，包括但不限于：
  - 所有界面文案（按钮、标题、提示语、设置项等）
  - App Store 的名称、描述、截图文案（后续单独准备）
  - 日期、时间格式需根据系统语言/地区自动适配
- 使用 SwiftUI 标准的 **Localizable.strings（或 String Catalog / .xcstrings）** 机制实现国际化，不要硬编码文案在代码中
- 语言跟随系统设置自动切换，无需额外设置项（除非用户主动要求覆盖系统语言，MVP 阶段不做）
- 文案风格需与"复古文艺"定位一致：中文用词典雅但不生僻，英文表达简洁、有质感，避免过于口语化或生硬的技术感词汇（例如避免直接用 "Delete Entry"，可考虑更柔和的措辞，具体文案后续可再打磨）
- 开发时建议从项目搭建阶段就引入本地化字符串文件，避免后期批量替换硬编码文本

**语感原则**：
- 中文避免使用"删除""确定""取消"这类纯功能性词汇的生硬直用，可用"抹去""留下"等更有画面感的措辞，但**核心操作类按钮需保留清晰可辨识的备选词**（如"抹去"旁可小字注明含义明确的次要提示），避免用户误解操作后果
- 英文避免过度花哨的诗意堆砌，保持简洁但略带温度，如 "Quietly saved" 优于 "Auto-saved" 的生硬感，也优于过长的诗化句子
- 涉及**不可逆操作**（如删除）时，文案可以柔和，但必须让用户清楚意识到后果，不能因为追求文艺而牺牲可用性和安全感
- 以上文案为参考基调，实际开发中可根据具体控件空间、系统组件规范做长度调整

## 视觉设计：App 图标与字体

### App 图标风格
- **整体基调**：复古文艺、温润克制，避免明亮饱和的"科技感"配色（不要用鲜艳渐变、玻璃拟态等潮流风格）
- **色彩方向**：低饱和度的暖色调为主，可参考——
  - 米白 / 牛皮纸色（信笺底色感）
  - 墨绿、赭石、暗棕、藏青（复古墨水瓶、皮质封面的颜色）
  - 点缀一点暖金色或朱红，作为细节提亮（如印章、烫金边）
- **图形意象**（任选其一或组合，避免堆砌过多元素）：
  - 一支简化的钢笔/羽毛笔，搭配一道墨迹或笔画
  - 摊开的笔记本/信笺一角，带一点卷边质感
  - 沙漏或日晷的简化剪影，呼应"拾光"的时间意象
  - 一枚简约的火漆印章造型
- **风格要求**：
  - 线条简洁，避免精细写实插画风（图标在小尺寸下需清晰可辨）
  - 可带轻微纹理感（如纸张肌理、做旧质感），但不能影响图标在深色/浅色模式下的辨识度
  - 需符合 Apple 图标规范（iOS/iPadOS 圆角矩形自动裁切、macOS 图标需符合"实体感"分层规范，参考 Apple HIG 的 App Icon 设计规范）
  - 避免使用文字（App 名称文字不放入图标内）
- **不要**：卡通风格、扁平几何图形拼贴、鲜艳撞色、写实照片风格

### 字体描述
App 内整体排版应传达"书写感、纸质感"，同时保证长文字阅读的舒适度。

- **界面 UI 字体（导航栏、按钮、列表等系统层级文字）**：
  - 中文：优先使用系统字体 **苹方（PingFang SC）**，保证系统一致性与可读性；如想更有质感，可探索衬线风格的中文字体（如"思源宋体"的克制字重），但需注意小字号下的清晰度
  - 英文：优先使用系统字体 **San Francisco**（SF Pro），保持跨平台一致性和原生质感
  - UI 层级文字不建议使用过于装饰性的字体，保证功能可读性优先

- **日记正文书写与阅读字体**：
  - 这是体现"文艺复古"调性的重点区域，可考虑：
    - 中文衬线字体：**宋体类**字体（如系统自带的"儿地设" / "Songti SC"），带有纸质书信的质感
    - 英文衬线字体：**Georgia、New York（苹果自带衬线字体）** 或其他古典衬线字体，营造手写信笺的氛围
  - 正文默认字号需保证舒适阅读（建议 16-18pt 起，用户可在设置中调整字号）
  - 行间距适当放宽（1.4-1.6 倍行高），模拟纸质笔记本的书写呼吸感
  - 可在设置中提供 1-2 种字体切换选项（如"衬线 / 无衬线"），但 MVP 阶段可先固定一种精心选择的字体，不做过多自定义选项

- **整体排版原则**：
  - 字重不宜过粗，保持克制优雅
  - 深色模式下需重新调校字体颜色（避免纯黑/纯白，用米白、暖灰替代，减少刺眼感，呼应纸张质感）
  - 标题与正文需有明确的字号/字重层级区分，但不追求花哨的多级样式

## 技术栈
- **语言/框架**：Swift + SwiftUI
- **最低系统版本**：iOS 17 / iPadOS 17 / macOS 14（可根据实际开发时最新情况调整）
- **数据持久化**：SwiftData（或 Core Data，二选一，优先 SwiftData）
- **数据同步**：iCloud（CloudKit）自动同步，无需用户手动操作
- **架构**：MVVM，尽量保持简单，不引入不必要的第三方依赖
- **本地优先**：无网络也可正常使用，网络恢复后自动同步

## 核心功能（MVP，仅限以下范围）

### 1. 日记条目
- 新建、编辑、删除日记
- 每篇日记包含：日期时间（自动记录创建时间，可编辑）、正文文字（纯文本/富文本二选一，建议先用纯文本 + 基本样式如加粗/斜体）
- 每天可以写多篇日记（不限制一天一篇）
- 自动保存草稿，无需手动点击保存

### 2. 日记列表与浏览
- 按时间倒序展示所有日记（最新在前）
- 列表显示：日期、时间、正文预览（前几行）
- 点击进入查看/编辑完整内容
- 按月/按年分组浏览（类似日历分组或分组列表）

### 3. 搜索
- 按关键字全文搜索日记内容
- 搜索结果高亮显示匹配文字

### 4. 日历视图
- 提供日历入口，标记出有日记的日期
- 点击日期跳转到当天的日记

### 5. 安全锁定
- 支持 Face ID / Touch ID / 密码解锁 App
- 用户可在设置中开启/关闭锁定
- App 切到后台或锁屏后自动锁定

### 6. iCloud 同步
- 日记数据存储在本地，通过 iCloud（CloudKit）自动同步到用户其他设备
- 同步过程无需用户干预，处理好冲突合并（以最后编辑时间为准或提示用户）
- 离线可用，联网后自动同步

### 7. 基本设置
- 开启/关闭 App 锁定
- 选择日记默认排序方式
- 导出日记（纯文本或简单格式，导出到文件，方便备份，MVP 阶段可选）

## 明确不做的功能（避免过度设计）
- ❌ 不支持插入图片、附件、语音
- ❌ 不做标签、心情打分、天气记录等扩展元数据（除非后续明确要求）
- ❌ 不做社交分享、多用户协作
- ❌ 不做富文本编辑器的复杂排版（表格、多级列表等）
- ❌ 不引入非必要的第三方库

## 设计原则
- **简洁优先**：界面干净，专注写作，减少视觉干扰
- **原生体验**：充分利用 SwiftUI 和 iOS/macOS 系统组件，符合 Apple 人机界面指南（HIG）
- **隐私安全**：所有数据默认仅存储在用户自己的 iCloud 账户中，不涉及第三方服务器
- **跨平台一致性**：iPhone/iPad/Mac 保持核心体验一致，同时适配各平台的交互习惯（如 Mac 的菜单栏、快捷键；iPad 的分栏视图）

## 开发建议顺序
1. 搭建基础项目结构（SwiftUI + SwiftData + CloudKit 配置，同时配置好中英文本地化字符串文件）
2. 实现日记的增删改查（CRUD）+ 本地持久化
3. 实现日记列表与详情页
4. 接入 iCloud 同步
5. 实现搜索功能
6. 实现日历视图
7. 实现 Face ID / 密码锁定
8. 适配 iPad / Mac 平台交互细节
9. 基本设置页面
10. 测试、优化、打磨细节动画和体验
